## Как начать

1. Сделайте форк репозитория.
2. Создайте ветку от `main`:
   - `feat/<короткое-описание>` для новых возможностей
   - `fix/<короткое-описание>` для исправлений
3. Внесите изменения небольшими логическими шагами.
4. Добавьте/обновите тесты.
5. Откройте Pull Request по шаблону.

> [!TIP]
> Один PR = одна цель. Небольшие PR проще ревьюить и быстрее мержить.

## Локальная среда

```bash
git clone https://github.com/alexsvdk/gigaplexity-mcp
cd gigaplexity-mcp

python3 -m venv .venv
source .venv/bin/activate
pip install -e .
pip install pytest pytest-asyncio
```

Если вы меняете интеграционную логику:

```bash
export GIGACHAT_COOKIES="..."
pytest -m integration -s
```

## Стиль кода

- Python: 3.11+
- Линтинг: `ruff` (конфиг в `pyproject.toml`)
- Общие принципы:
  - Простые и короткие функции
  - Понятные имена переменных
  - Без лишней магии и скрытых сайд-эффектов

## Тесты

Минимальный чек перед PR:

```bash
pytest
```

Рекомендации:

- Для bugfix добавляйте тест, который воспроизводит баг.
- Для новых фич добавляйте позитивный сценарий и как минимум один edge-case.
- Не смешивайте рефакторинг и изменение поведения в одном коммите без необходимости.

## Формат коммитов и PR

Предпочтительный формат коммитов:

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `test: ...`
- `refactor: ...`
- `chore: ...`

PR должен содержать:

- Краткое описание проблемы и решения
- Что изменилось по файлам/модулям
- Как проверяли (команды, результат)
- Возможные риски и ограничения
